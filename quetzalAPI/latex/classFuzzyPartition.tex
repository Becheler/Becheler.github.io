\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Element$>$\newline
class Fuzzy\+Partition$<$ Element $>$}

Clusters are implicitly defined because of cluster merging (what would make unique I\+Ds complicated to use) 

Template class usefull for formalizing that elements of a set can simultaneously belong to various clusters.

Fuzzy partition are objects of interest when one wants to represent the fact that an element of a set can belong to several clusters. This can be represented by a matrix M(i,j), with clusters in line and elements in column, where the general term mu(i,j) represent the coefficient of membership of an element to a cluster. Coefficients memberships summed on all clusters are equal to 1. \begin{DoxyWarning}{Warning}
In the present implementation, Clusters I\+Ds are not memorized, because it would complicate fuzzy partition operations as merging clusters. So clusters are stored in the order given by their identifiers at construction, but the I\+Ds themselves are not stored.
\end{DoxyWarning}
